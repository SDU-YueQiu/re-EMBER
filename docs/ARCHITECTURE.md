# kEmber 项目架构

## 📁 目录结构

```
kEmber/
├── src/
│   ├── math/              # 数学基础层
│   │   ├── math256.h                 # 256位整数向量、行列式
│   │   └── plane_geometry256.h       # 平面、齐次坐标
│   │
│   ├── geometry/          # 几何图元层
│   │   ├── geometry256.h/cpp         # Line256, Segment256, Polygon256
│   │
│   ├── algorithm/         # 核心算法层
│   │   ├── clipping.h/cpp            # 多边形裁剪
│   │   ├── bsp.h/cpp                 # BSP树
│   │   ├── winding_number.*          # (待实现) 环绕数系统
│   │   └── subdivision.*             # (待实现) 空间细分
│   │
│   ├── core/              # 顶层接口
│   │   └── bool_problem.h/cpp        # 布尔运算管理器
│   │
│   ├── tests/             # 测试代码
│   │   └── math256_tests.h/cpp
│   │
│   └── main.cpp           # 程序入口
│
├── include/               # 第三方库
│   └── slimcpplib/                   # 256位整数库
│
├── reference/             # 论文文档
└── docs/                  # 项目文档
```

## 🔗 依赖关系

```
main.cpp
  └─> tests/
       └─> algorithm/
            └─> geometry/
                 └─> math/
                      └─> slimcpplib/
```

**设计原则**：低层不依赖高层，保持单向依赖。

## 📊 当前进度

| 步骤 | 模块 | 状态 | 备注 |
|------|------|------|------|
| **a 数学基础** | `math256`, `plane_geometry256` | ✅ 完成 | 256位整数、平面交点、分类谓词 |
| **b 递归空间细分** | `subdivision` | ✅ 基本完成 | 分割策略（如重心分离）待优化 |
| **c 叶节点提前剪枝** | `clipping` + `bsp` + 剪枝逻辑 | ⏳ 进行中 | BSP与多边形裁剪已完成；剪枝逻辑（基于WNV和指示函数）正在实现 |
| **d 环绕数追踪体系** | `winding_number` | ✅ 基本完成 | 线段追踪、WNV传播、参考点更新已完成；边缘情况（路径退化、共面）鲁棒性待完善 |
| **e 输出筛选** | `bool_problem` | ⏳ 初步集成 | 指示函数映射、多边形保留/反转/丢弃已实现，需与上游完整对接 |

**整体状态**：最小可执行流水线已跑通。当前工作聚焦于：
- 优化：分割策略、并行化、早期剪枝
- 鲁棒性：边缘情况处理（射线恰过顶点/边、共面重叠）、输入检查
